🔐 인증 흐름

4. 전략 시나리오 STRATEGY

CSAP 인증 획득을 위한 두 가지 경로

⭐ 핵심 분기점: "Zadara 본사가 한국 CSAP 인증 추진에 협조할 의향이 있는가?"

✅ 시나리오 A — 본사 협조 가능 (권장)

  • Zadara 인프라 그대로 + 한국 포털 → CSAP 취득
  • 기간: 1~1.5년 | 인원: 5~8명

🔄 시나리오 B — 본사 협조 어려움

  • KT/NHN Cloud CSAP 인증존 활용
  • 기간: 1.5~2년 | 인원: 8~12명

인증 관련 정리 수정됨

❌ CC인증 — 해당 없음 (제거)

CC인증은 보안 장비/OS용 인증. SaaS 포털에는 CSAP가 적합. 포트폴리오에서 제거 필요.

✅ CSAP (클라우드 보안 인증제) — 목표 인증

  • 전자정부 프레임워크: 불필요 (SaaS 서비스라 자유 기술 스택 가능)
  • 서버실: 불필요 (클라우드 배포로 해결)
  • 2차 미팅 확인 필요: "CSAP vs 보안 기능 확인서 — 어느 것인가?"
