1. 프로젝트 배경 & 목표 CONTEXT

이 프로젝트가 왜 시작되었고 무엇을 달성하려 하는가

🎯 핵심 목표

  • Zadara Cloud 인프라를 기반으로 한국형 웹 포털을 자체 개발
  • 한국 공공시장 진출을 위한 CSAP 인증 획득
  • 조달청 디지털서비스몰 등록을 통한 공공납품 실현

🏢 회사 구조

  • Zadara 본사 (이스라엘계 글로벌 클라우드)
  • 한국 지사 — 포털 구축 주도
  • 대상 인프라: compute-iaas-kr1-01.zadara.com

👤 내 포지션

  • 직함: CTO / 기술이사 (Korea)
  • 연봉: 7,000만원 (협의됨)
  • 역할: 개발 설계·감독, 전략 기획, 보안 인증 총괄
  • 비고: 업무 범위 대비 연봉 상향 여지 있음

2. Zadara 기술 분석 결과 ANALYSIS DONE

면접 후 직접 시스템에 접속하여 분석 완료 (19개 페이지, 60+ API 엔드포인트)

📌 플랫폼 구조

🖥 Frontend

  • Vue.js 기반 SPA
  • Chart.js / ECharts (모니터링 차트)
  • Socket.IO 실시간 스트리밍 확인
  • 이벤트 로그: auth.user_logged_in, compute.instance.create.end 등

⚡ Backend API

  • REST API: /api/v2/ 구조
  • OpenStack Keystone 인증 (2단계)
  • Opaque Token 방식 (JWT 아님)
  • X-Auth-Token 헤더로 API 호출

🔐 인증 흐름

1. POST /api/v2/identity/auth → Domain Token 발급
2. GET /api/v2/identity/users/myself/projects → 프로젝트 목록
3. POST /api/v2/identity/auth → Project Token 발급 (2단계)
4. 이후 모든 API: Header: X-Auth-Token: {token}
5. 토큰 만료 → 자동 갱신 로직 필요

3. 전략 시나리오 STRATEGY

CSAP 인증을 위한 두 가지 경로

✅ 시나리오 A — 본사 협조 가능 (권장)

  • Zadara 인프라 그대로 활용
  • 한국 포털 개발 → CSAP 취득
  • 가장 빠르고 강력한 경로
  • 기간: 약 1~1.5년
  • 인원: 5~8명

🔄 시나리오 B — 본사 협조 어려움

  • KT Cloud G / NHN Cloud CSAP 인증존 활용
  • Zadara는 백엔드 API로만 연동
  • 기간: 약 1.5~2년
  • 인원: 8~12명

⭐ 전략의 핵심 분기점: "Zadara 본사가 한국 CSAP 인증 추진에 협조할 의향이 있는가?"

4. 만들어야 할 문서 목록 TODO DOCUMENTS

계획 실현을 위한 단계별 필요 문서 체계

📋 1단계 — 사업 기획 문서 (지금 당장)

  • ✅ 제안서 (Proposal) — 슬라이드로 상당 부분 완성됨
  • ⬜ 사업계획서 — 시장 규모, 수익 모델, 투자 비용, 손익 분기점
  • ✅ 기술 분석 보고서 — Zadara API 분석 결과 (HTML 완성)
  • ⬜ 파트너십 협약서 초안 — Zadara 본사와의 협력 조건 정의

🏗 2단계 — 개발 설계 문서 (개발 착수 전)

  • ⬜ 요구사항 정의서 (SRS) — 시스템이 무엇을 해야 하는지 명세
  • ⬜ 시스템 아키텍처 설계서 — 프론트/백엔드/DB/인프라 구조도
  • ⬜ API 설계서 — 우리 포털의 내부 API 명세 (Swagger 형식)
  • ⬜ DB 설계서 — 테이블 구조, ERD
  • ⬜ UI/UX 와이어프레임 — 주요 화면 레이아웃

🔒 3단계 — 보안/운영 문서 (CSAP 인증 전 필수)

  • ⬜ 정보보호 정책서 — 회사의 보안 원칙 선언문
  • ⬜ 개인정보 처리방침 — 법적 필수, 서비스에 게시
  • ⬜ 접근통제 정책서 — 권한별 시스템 접근 기준
  • ⬜ 암호화 정책서 — 데이터 암호화 기준 및 방식
  • ⬜ 취약점 점검 보고서 — 보안 스캔 결과 및 조치 내역
  • ⬜ 재해복구(DR) 계획서 — 장애 시 복구 절차
  • ⬜ 보안 사고 대응 절차서 — 침해 발생 시 대응 프로세스

📑 4단계 — CSAP 신청 문서

  • ⬜ 서비스 설명서 — 서비스 구조, 데이터 흐름도 포함
  • ⬜ 보안 통제 항목 이행 증적 — 각 보안 항목별 조치 완료 증거
  • ⬜ 클라우드 인프라 구성도 — 실제 배포 환경 다이어그램
  • ⬜ SLA(서비스 수준 협약서) — 가용성, 장애 대응 시간 기준

💼 5단계 — 영업/계약 문서 (공공 납품 시)

  • ⬜ 서비스 소개서 — 1~2페이지 영업용 자료
  • ⬜ 표준 이용약관
  • ⬜ 표준 계약서
  • ⬜ 조달청 등록 신청서
